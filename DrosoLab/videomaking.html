User Manual: Image Sequence to Video Converter

1. Overview & Purpose üéûÔ∏è

The Image to Video Converter is a utility designed to compile a sequence of still images from an experiment into a single video file. Its primary purpose is to aid in ground truthing‚Äîthe process of manually verifying the results of an automated tracking system.

The key feature of this tool is its ability to automatically read the timestamp from each image's filename and overlay it onto the video frame. This allows a researcher to watch the generated video and visually confirm that the events recorded in the data logs correspond to the correct time and visual evidence, ensuring the accuracy and reliability of the automated analysis.

2. The User Interface: A Step-by-Step Guide üñ•Ô∏è

The application provides a simple, linear workflow to guide you through the video creation process.

  1. Select Image Folder:

    Click the "Browse" button to open a file dialog. Navigate to and select the folder that contains the sequence of images (e.g., the captured or processed folder from an experiment) you wish to convert. The path to the selected folder will appear in the text box above the button.

  2. Set Frames Per Second (FPS):

    Use the spin box to set the playback speed of the output video. The FPS value determines how many images (frames) are displayed each second. A higher value will result in a faster, smoother video, while a lower value will create a slower, more deliberate playback.

  3. Select Output Video:

    Click the "Select Output" button to choose a location and name for the final video file. The file will be saved in the .mp4 format, which is widely compatible. If you do not add the extension, it will be added automatically.

  4. Create Video:

    Once you have configured the inputs and output, click the "Create Video" button. The application will begin processing the images and compiling the video. A success message will appear once the process is complete.

3. The Conversion Process: How It Works ‚öôÔ∏è

The tool performs two critical operations in the background to ensure the video is accurate and useful for ground truthing.

3.1. Chronological Sorting

Instead of simply reading files in alphabetical order, the application intelligently sorts them by time. It uses a pattern matching system (regular expressions) to find the date and time embedded in each filename. For example, it recognizes filenames like:

image_20250719_004534.png

It extracts the timestamp (20250719_004534), converts it to an actual date and time, and then sorts the entire list of images chronologically. This guarantees that the video frames are assembled in the correct temporal sequence, accurately reflecting the timeline of the experiment.

3.2. Timestamp Overlay

This is the core feature for ground truthing. For each image in the sorted list, the tool performs the following steps before adding it to the video:

  1. The timestamp is extracted from the filename.
  2. It is converted into a human-readable format (e.g., "19 July 2025 00:45:34").
  3. This text is rendered onto a semi-transparent black background for maximum readability against any image content.
  4. The final timestamp is rotated 90 degrees and placed vertically on the right-hand side of the frame. This ensures the timestamp is clearly visible without obstructing the central area of the image where the experiment takes place.

4. The Output File üé¨

The final output is a standard MP4 video file. Each frame of the video corresponds to one of the source images, now enhanced with a clear, legible timestamp overlay. This video can be played in any standard media player, allowing for easy review, scrubbing, and frame-by-frame analysis to visually validate the results of your automated tracking data.
